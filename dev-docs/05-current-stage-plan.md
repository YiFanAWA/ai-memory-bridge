# 当前阶段实施真源 — v0.2.0 → v0.3.0

## 阶段目标
**"轻量体验优化"** — 不改架构，不改数据模型，只做面板交互优化。

## 子阶段 (Sub-stages)

### S1: 面板搜索框 + 排序 🟢 轻量
- **目标**: 用户可在侧栏面板内搜索记忆、切换排序方式
- **改动**: MemoryPanel.ts, styles.css
- **验证**: 
  1. 在面板搜索框输入关键词 → 列表实时筛选
  2. 点击排序按钮 → 列表按名称/时间/tags 重新排列
  3. 搜索框为空时显示全部
- **不做什么**: 不改 bridge 格式、不改 MCP 工具、不改后端搜索
- **风险**: 低（仅 UI 层改动）

### S2: 一键复制 MCP 配置 🟢 轻量
- **目标**: 面板内按钮一键复制 Claude Code MCP JSON 到剪贴板
- **改动**: MemoryPanel.ts (新增按钮 + clipboard API)
- **验证**:
  1. 点击"复制配置"按钮 → 剪贴板内容正确
  2. 粘贴到 ~/.claude/mcp.json → 格式合法
- **不做什么**: 不自动写入 mcp.json
- **风险**: 低

### S3: 首次使用引导 🟢 轻量
- **目标**: 首次打开面板时显示 3 步引导（拖拽→启动 MCP→配置 Claude）
- **改动**: MemoryPanel.ts + styles.css
- **验证**:
  1. 全新安装 → 面板显示引导
  2. 已有记忆 → 不显示引导
  3. 引导可关闭
- **不做什么**: 不修改数据迁移逻辑
- **风险**: 低

### S4: 记忆分组（按 tags）🟡 标准
- **目标**: 面板内按 frontmatter tags 自动分组，支持折叠/展开
- **改动**: MemoryPanel.ts (UI) + bridge 索引加 tags 字段
- **验证**:
  1. 添加带不同 tags 的笔记 → 面板自动分组
  2. 点击分组标题 → 折叠/展开
  3. 无 tags 的笔记归入"未分类"
- **不做什么**: 不创建新的数据存储，不改 MemoryStore 数据模型
- **风险**: 中 — 改 bridge 格式，需向后兼容

## 阶段验收标准

- [ ] S2: 一键复制输出合法的 MCP 配置 JSON
